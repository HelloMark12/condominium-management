Preferred approach:

* Add a dedicated test database setup script.
* Run drizzle-kit push for the base schema.

* Use only the isolated test DATABASE_URL.
* Never modify development or production data.
* Run non-interactively.
* Stop immediately if the migration fails.
* Work without Clerk secrets.
