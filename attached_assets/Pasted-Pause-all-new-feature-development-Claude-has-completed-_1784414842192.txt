# PRIORITY 1 — BILLING ACCURACY

## 1. Fix current-count versus monthly-peak billing

In:

artifacts/api-server/src/routes/units.ts

inside updateMonthlyUsage(), the tier and estimated amount are currently calculated using currentActiveCount.

This is incorrect.

The current billing month must always use the highest number of simultaneously active billable apartments reached during that month.

Correct behaviour:

1. Calculate or atomically update the monthly peak.
2. Obtain the resulting peak value.
3. Pass the monthly peak—not the current count—to:

   * calculateTier()
   * calculateEstimatedAmountCents()
4. Store the resulting tier and estimate in monthly_usage_records.
5. Ensure any company-level displayed tier is also based on the monthly peak or remove that duplicate source of truth.

After a company reaches a peak of 10 apartments and later archives down to 2:

* active apartment count = 2
* monthly peak = 10
* current-month tier remains based on 10
* current-month estimate remains based on 10
* GET /companies/:id/usage must return the peak-based tier and estimate
* GET /auth/me must return the same peak-based administrator plan
* GET /companies/:id/subscription must agree with both

Add a regression test that performs this complete workflow through the real API and database.

Do not merely test calculateTier() as a helper.
